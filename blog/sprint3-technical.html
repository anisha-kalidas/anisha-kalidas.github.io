JavaScript & The DOM

1. An analogy to describe HTML and CSS

If we think of a website as being a house. A house has rooms, and each room contains furniture and appliances. Similarly, a webpage has sections (e.g. header, body, footer), and each section contains images and text. HyperText Markup Language (HTML) specifies how content is structured on the page, and organises and presents this content in a structured hierarchy of elements. If we apply the house analogy to this hierarchy, we could say that HOUSE is the parent element of LIVING_ROOM is the parent element of COUCH and RUG, which are sibling elements.

However, not all rooms, couches, and rugs look the same, nor do words or images on a page. That’s where CSS (Cascading Style Sheets) comes in – CSS is a set of styling instructions which define how elements look on the page. It describes their color, font, size, position, shape, and more. So using CSS we are able to change the colour and fabric of the couch and the size and texture of the rug and decide where in the room to place them both.

2. Control Flow & Loops

Control flow is the order in which statements or functions are executed. In JavaScript, we read the code starting from the first line until the last line, unless in the code there is some statement that changes that control flow (such as a loop). We can think of control flow like a recipe for baking cupcakes. We start at the top and follow the instructions until we get to the bottom of the recipe.

Loops in JavaScript are used to execute the same block of code a given number of times, or while a specified condition is true. They are handy if you want to run the same code over and over again. Now, say we wanted to ice all of our cupcakes. Instead of having to say:

  1. Spread a spoonful of icing on to a cupcake
  2. Spread a spoonful of icing on to the next cupcake
  3. Spread a spoonful of icing on to the next cupcake
  4. Spread a spoonful of icing on to the next cupcake
  5. Spread a spoonful of icing on to the next cupcake
  6. Spread a spoonful of icing on to the next cupcake
  7. Spread a spoonful of icing on to the next cupcake
  8. Spread a spoonful of icing on to the next cupcake
  9. Spread a spoonful of icing on to the next cupcake...

We can write a loop that says: starting with the first cupcake and continuing for as many cupcakes as there are, take a spoonful of icing and spread it on top. Then move to the next cupcake. When you run out of cupcakes, stop. This is much more efficient, saves time and is easier to adjust if we want to change the process! 

3. What is The DOM ?

Every page that we view in our browser is a document. The Document Object Model (DOM) is a tree-like structure, a sort of map, made up of all the elements that exist in the HTML document. In the DOM, every HTML-tag is an object, all of these objects are called ‘nodes’ and all of these nodes are accessible using JavaScript.

Elements in the DOM are arranged in a hierarchy that defines what you eventually see in the browser. We can think of it like a family tree. In the example below, [div] and [script] are children of the [body] element. Because they share the same parent, this makes them siblings. This hierarchy is used to help organise the HTML elements. It is also used to help CSS rules make sense of which styles to apply to certain elements.

Whether we are using JavaScript to change an element's text, swap out an image with a different one, move an element from one part of your document to another, set an inline style, or perform any other change we want to make, we will be modifying the DOM. 

4. The difference between accessing data from arrays and objects

Think of an array as a list of similar items, for example animals. We can select items in this array by their index - indexes start at zero. So for an array where animals = [cat, dog, bird]:

  • animals[0] = cat
  • animals[1] = dog
  • animals[2] = bird

Typically for arrays we use bracket notation.

An object is similar to an array but each object has a set of properties. So for the cat object we might have:

  • colour: “ginger”
  • legs: 4
  • tail: true
  • eye colour: “green”

Instead of selecting a value by index, we use a key, for example; cat.colour = ginger and cat.legs = 4

Typically for objects we use dot notation, however there are some limitations. Dot notation will work only when the property name is alphanumeric ( _ and $ are allowed ), does not start with a number, and does not contain any spaces. In circumstances where these requirements are not met, we resort to using bracket notation. For example: cat[“eye colour”] = green.

5. What are Functions ?

A function is a block of code designed to perform a particular task. It is reusable code which can be called anywhere in our program. This makes our lives much easier as it eliminates the need for writing the same code again and again. A function can take one or more parameters and use these to complete a function.
